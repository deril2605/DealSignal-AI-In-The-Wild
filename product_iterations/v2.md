# DealSignal MVP v2

## Working Title

Lead scoring.

## Goal

Move from "detect meaningful change" to "rank the most worthwhile opportunities for human review."

`v1` tells the user what changed.

`v2` should tell the user what deserves attention first.

## Product Thesis

By `v1`, the system can:

- collect signals
- detect deltas
- mark some changes as alerts

The next step is prioritization.

Not every alert is equally valuable. `v2` should assign a score that helps a user quickly focus on the most relevant signals across the watchlist.

## Desired User Outcome

A user should be able to open the system and quickly answer:

- what are the top opportunities this week?
- why are these ranked above others?
- which should I review first?

## Proposed Scoring Inputs

### 1. Signal Strength

How strong or explicit is the underlying strategic intent?

Examples:

- "we are actively exploring acquisitions" -> very strong
- "we are expanding in a new geography" -> strong
- "we are excited about AI" -> weak

This should likely be one of the strongest score contributors.

### 2. Recency

More recent changes should rank higher than stale ones.

This should remain a meaningful but not dominant feature.

### 3. Frequency / Reinforcement

Signals that appear across multiple independent sources should receive a boost.

This should reward corroboration, not low-value repetition.

### 4. Fit To Firm Thesis

Events that match the user or firm's target themes should score higher.

This is where the system becomes more opinionated and useful.

Examples:

- strong fit to target sectors
- strong fit to strategic themes
- weak fit despite interesting signal

### 5. Relationship Graph / Warm Connection

If CRM or relationship data indicates a warm path, score should increase.

This should be treated as an optional bonus, not a hard dependency for `v2`.

### 6. Source Quality

Signals from higher-trust sources should generally rank above weaker or noisier sources.

Examples:

- earnings calls
- filings
- Reuters / Bloomberg
- company blog / PR

## Proposed v2 Output

The output should be a ranked list of opportunities, not just a list of alerts.

Examples:

- top 10 weekly suggestions
- top companies by current score
- explanation of why each suggestion ranked highly

## Proposed Scoring Model

First-pass idea:

`lead_score = change_significance + signal_strength + recency + reinforcement + thesis_fit + relationship_bonus + source_quality`

The exact weights should remain configurable or easy to tune.

## Proposed Data Model Additions

### `lead_scores`

One row per scored opportunity or scoring snapshot.

Suggested fields:

- `company_id`
- `source_event_id`
- `narrative_delta_id`
- `lead_score`
- `signal_strength_score`
- `recency_score`
- `reinforcement_score`
- `thesis_fit_score`
- `relationship_score`
- `source_quality_score`
- `explanation`
- `created_at`

### Optional: `thesis_profiles`

If thesis fit becomes configurable, add a table or config structure for:

- target themes
- target sectors
- preferred geographies
- exclusions

## Pipeline Extension

After `v1` delta evaluation:

1. identify candidate events or alert-worthy deltas
2. calculate lead-score components
3. produce final lead score
4. persist explanation and component breakdown
5. rank in UI and digest/report views

## UI Changes For v2

Potential additions:

- "Top Opportunities" module on home page
- ranked weekly suggestions page
- score breakdown per opportunity
- filters for thesis fit, source quality, and relationship bonus
- clearer "why this ranked high" explanation

## Success Metric

Primary:

- top 10 weekly suggestions are worth a quick human look

Better operational version:

- a majority of top-ranked suggestions feel relevant enough for review

## Risks

- scores may feel arbitrary if component reasoning is hidden
- over-weighting recency can surface noisy short-term signals
- over-weighting frequency can reward PR repetition
- thesis fit can become too rigid if implemented poorly
- CRM integration can add complexity and data quality problems

## Suggested Implementation Order

1. add lead-score model/table
2. implement rule-based scoring with clear components
3. expose score breakdown in UI
4. validate rankings on real weekly runs
5. add optional relationship/CRM bonus later if needed

## Non-Goals For v2

- full CRM graph intelligence on day one
- perfect scoring weights
- autonomous outreach
- outbound workflow automation

## Decision Standard

`v2` is successful if the system consistently places the most review-worthy items near the top and gives a user enough explanation to trust the ranking.
